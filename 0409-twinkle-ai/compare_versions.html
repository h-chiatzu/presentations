三種方法的優劣勢分析

23%
ES 多輪搜尋版（gemma-3-4B + 手動 agent 迴圈）
5 正確 / 6 部分 / 10 錯誤 | 平均 1.8 輪 | 3.8s/題

優勢

  • 多輪搜尋策略 — 最多 3 輪，資料不足時自動追加查詢
  • 關鍵字多元 — 用 | 分隔多組詞組，覆蓋不同切入角度（如 Q08 同時搜台積電跌幅、外資賣超、航運類股）
  • 誠實拒答 — 找不到資料時會說「無法回答」，不捏造（Q05、Q19、Q21）

劣勢

  • 回答發散 — 常列出大量周邊資訊卻遺漏核心關鍵字
  • 多輪未提升準確率 — 有時反而帶入雜訊（Q06 查到 2017 年舊資料）
  • 新事件覆蓋率差 — 2025-2026 年事件錯誤率達 45%（10/22 題）
45%
Function Call 版（gemma-3-4B + Ollama tool use）
10 正確 / 4 部分 / 7 錯誤 | 平均 1.0 輪 | 7.7s/題

優勢

  • 回答結構化 — 平均 369 字，附資料來源與日期，易於核查
  • 搜尋廣度高 — 能撈出相關背景資訊，部分正確題數最多（10 題）
  • 單輪查詢精準 — 單輪完成全部 22 題，設計簡潔
  • 專業題表現佳 — 如 Q05 完整引用鄭明典的逐點說明

劣勢

  • 幻覺率高 — 資料不足時傾向「補充生成」而非拒答（Q14 捏造 16 面獎牌、Q18 答成中國 GDP、Q21 混淆 NASA 任務）
  • 耗時最長 — 7.7s/題，準確率未因此提升
  • 數字不穩 — Q02 就職時間、Q03 地震規模均答錯
100%
CNA MCP（Claude Opus + MCP Server）
22 正確 / 0 部分 / 0 錯誤 | 平均 ~1.3 輪 | ~2s/題

優勢

  • 正確率最高 — 錯誤率為零，所有答案直接命中關鍵事實
  • 回答精簡 — 平均 60 字，無贅字
  • 時效性最強 — 正確涵蓋 2025-2026 年最新動態（Q12、Q17、Q20）
  • 數字精準 — 規模、傷亡數、關稅稅率、獎牌數均正確

劣勢

  • 回答過短 — 缺乏背景說明與來源引用，無法自我核查
  • 版本差異難判 — Q01 經濟效益數字與標準答案不同（23 兆 vs 10.5 兆）
  • 非主流事件覆蓋弱 — Q11 亞太棒球場漏掉全中運開幕
